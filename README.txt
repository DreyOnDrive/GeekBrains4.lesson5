Напишем пользовательскую историю, описывающую функции сервиса:
Как пользователь я хочу от сервиса следующие фукции:
1. Иметь возможность создание продукта в категории "Food"
2. Иметь мозможность получить информацию о определённом продукте
3. Иметь возможность изменить данные об определённом продукте
4. Иметь возможность удалить определённый продукт

Техническая реализация сервиса:
1.  Создание продукта реализуется командой POST, в теле которой передаётся три поля:
        Title - название продукта
        CategoryTitle - название категории
        Price - цена
    В от сервиса приходит ответ тело с четырьмя полями, помимо технических кодов ответа:
        Title - название продукта
        CategoryTitle - название категории
        Price - цена
        Id - уникальный номер продукта

2.  Получение продукта по уникальному номеру реализуется командой GET, в параметрах которой передаётся одно поле:
        Id - уникальный номер продукта
    В от сервиса приходит ответ тело с четырьмя полями, помимо технических кодов ответа:
        Title - название продукта
        CategoryTitle - название категории
        Price - цена
        Id - уникальный номер продукта

3.  Изменение продукта по уникальному номеру реализуется командой PUT, в параметрах которой передаётся четыре поле:
        Title - название продукта
        CategoryTitle - название категории
        Price - цена
        Id - уникальный номер продукта
    В от сервиса приходит ответ тело с четырьмя полями, помимо технических кодов ответа:
        Title - название продукта
        CategoryTitle - название категории
        Price - цена
        Id - уникальный номер продукта

4.  Удаление продукта по уникальному номеру реализуется командой DELETE, в параметрах которой передаётся одно поле:
        Id - уникальный номер продукта
    В от сервиса приходит только технический код ответа.
